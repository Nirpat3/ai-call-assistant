To access and pull a list of contacts from Android and iOS, the approach differs significantly due to OS-level privacy restrictions. Below is a practical, production-ready breakdown—including what’s allowed, what’s not, and how real apps do it.

⸻

1. Core Reality (Important)
	•	You CANNOT silently access contacts on either platform
	•	Explicit user permission is mandatory
	•	Apple is much stricter than Android
	•	Contacts access is read-only unless user allows write

⸻

2. iOS (iPhone / iPad)

✅ Allowed
	•	Access contacts only after user permission
	•	Read selected or full contact list
	•	Sync to backend only if disclosed in Privacy Policy

❌ Not Allowed
	•	Background harvesting
	•	Auto-upload without user action
	•	Access without App Store–declared usage

⸻

iOS Implementation (Swift)

Step 1: Add Permission Description (Info.plist)

<key>NSContactsUsageDescription</key>
<string>This app accesses contacts to help you connect with people you know.</string>


⸻

Step 2: Request Permission & Fetch Contacts

import Contacts

let store = CNContactStore()

store.requestAccess(for: .contacts) { granted, error in
    if granted {
        let keys = [
            CNContactGivenNameKey,
            CNContactFamilyNameKey,
            CNContactPhoneNumbersKey,
            CNContactEmailAddressesKey
        ]

        let request = CNContactFetchRequest(keysToFetch: keys as [CNKeyDescriptor])

        try? store.enumerateContacts(with: request) { contact, _ in
            print(contact.givenName, contact.phoneNumbers)
        }
    }
}


⸻

iOS Best Practice
	•	Use Contacts Picker UI if you only need selected contacts
	•	Hash phone numbers before uploading
	•	Explain purpose clearly during permission prompt

⸻

3. Android

Android allows more flexibility, but still permission-gated.

⸻

Required Permissions (AndroidManifest.xml)

<uses-permission android:name="android.permission.READ_CONTACTS"/>


⸻

Request Runtime Permission (Android 6+)

if (ContextCompat.checkSelfPermission(this, Manifest.permission.READ_CONTACTS)
    != PackageManager.PERMISSION_GRANTED) {
    ActivityCompat.requestPermissions(this,
        new String[]{Manifest.permission.READ_CONTACTS}, 101);
}


⸻

Query Contacts (Kotlin)

val cursor = contentResolver.query(
    ContactsContract.CommonDataKinds.Phone.CONTENT_URI,
    null, null, null, null
)

while (cursor?.moveToNext() == true) {
    val name = cursor.getString(
        cursor.getColumnIndex(ContactsContract.CommonDataKinds.Phone.DISPLAY_NAME)
    )
    val number = cursor.getString(
        cursor.getColumnIndex(ContactsContract.CommonDataKinds.Phone.NUMBER)
    )
    println("$name : $number")
}
cursor?.close()


⸻

Android Best Practice
	•	Normalize phone numbers (E.164)
	•	Avoid pulling photos unless needed
	•	Do not sync automatically without user confirmation

⸻

4. React Native / Cross-Platform

Use Library

npm install react-native-contacts

Fetch Contacts

import Contacts from 'react-native-contacts';

Contacts.requestPermission().then(permission => {
  if (permission === 'authorized') {
    Contacts.getAll().then(contacts => {
      console.log(contacts);
    });
  }
});

⚠️ Apple will reject apps that:
	•	Use contacts for marketing
	•	Upload without transparency
	•	Lack privacy disclosure

⸻

5. Web Apps (VERY LIMITED)

❌ Browsers CANNOT access contacts directly

✅ Workarounds

Method	Supported
Google Contacts OAuth	Android/Web
iCloud CSV Import	Manual
File upload (.vcf)	Yes


⸻

6. Common Production Patterns (How Big Apps Do It)

Invite Flow (WhatsApp, Uber)
	1.	Ask permission
	2.	Hash phone numbers
	3.	Match against existing users
	4.	Discard unmatched numbers

Enterprise / POS Use
	•	User-initiated export only
	•	Explicit “Sync Contacts” button
	•	Audit logging

⸻

7. App Store / Play Store Compliance (Critical)

Apple Requires
	•	Privacy Policy disclosure
	•	App Tracking Transparency if combined with analytics
	•	Purpose limitation

Google Requires
	•	Justification if contacts are core to app
	•	Data Safety form declaration

⸻

8. Recommended Architecture (Secure)

Device Contacts
   ↓ (Permission)
Local Filter / Hash
   ↓
Secure API
   ↓
Temporary Processing
   ↓
Delete Raw Data


⸻

9. What You Should NOT Do (Guaranteed Rejection 🚫)
	•	Pull contacts on app install
	•	Use for cold marketing
	•	Store raw contacts indefinitely
	•	Share with third parties
